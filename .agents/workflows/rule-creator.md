# RuleCreator: ルール作成ワークフロー

プロジェクトのルートにある `GEMINI.md` または `.agents/rules/` に、リポジトリ固有のルールを迅速かつ適切に追加する。

## ルールの配置場所
- **インデックス**: `GEMINI.md` (プロジェクトルート)
- **詳細ルール**: `.agents/rules/` 内の各 Markdown ファイル

## ルール記述のガイドライン
- **簡潔さ**: 長文を避け、箇条書きや短い文で記述する。
- **レポ固有**: 汎用的なツール（git, npm, tsc 等）の一般的な使い方は書かず、プロジェクト固有の規約、ディレクトリ構造上の制約、特定ライブラリの癖などに絞る。
- **言語**: すべて日本語で記述する。

## 手順
1. **ヒアリング**: ユーザーに対し、「ルール作成のためのヒアリング項目」に基づいて質問する。
2. **ドラフト作成**: 指示を極限までシンプルにまとめ、追記内容（または新規ファイル作成）を提案する。
3. **適用**: ユーザーの承認後、ファイルを更新する。

## ルール作成のためのヒアリング項目
- **ルールの対象**: どの機能、ディレクトリ、またはフローに関するルールか？
- **具体的な制約**: 「プログラミング上の禁止事項」や「必ず守るべき手順」は何か？
- **既存ルールとの関係**: 既存のルールと競合しないか？
- **期待される動作**: AI がそのルールを読み取ったとき、具体的にどのような行動をとってほしいか？
