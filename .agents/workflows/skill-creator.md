# SkillCreator: スキル作成ワークフロー

新しい専門スキルを効率的に定義し、プロジェクトに適した形式で `SKILL.md` を作成する。

## 手順

1. **ヒアリング**: ユーザーに対し、「スキル作成のためのヒアリング項目」に基づいて質問を行い、必要な情報を収集する。
2. **設計**: 収集した情報を元に、スキルのスコープ、指示（Instructions）、および構成を設計する。
3. **ドラフト作成**: 下記のテンプレートをベースに、`SKILL.md` の下書きを作成する。
4. **レビューと修正**: ユーザーに下書きを確認させ、フィードバックに基づいて調整する。
5. **最終化**: `.agents/skills/<skill-name>/SKILL.md` を作成し、必要に応じて補助ファイル（scripts, examples 等）の構成を提案する。

## スキル作成のためのヒアリング項目

新しいスキルを作成する際、必ず以下の点を確認する。
- **スキルの名称**: エージェントが認識しやすい簡潔な名前。
- **主目的**: そのスキルは何を解決するためのものか？
- **主な利用シーン**: どのようなタスクの時にこのスキルを読み込むべきか？
- **期待されるアウトプット**: スキル適用後の具体的な成果物（例：コード、ドキュメント、テスト結果）。
- **従うべきルール/制約**: 特定のライブラリの使用禁止、命名規則、セキュリティルールなど。
- **参照情報**: 既存のコード、API リファレンス、過去の成功事例など。

## SKILL.md のテンプレート

新しい `SKILL.md` を作成する際は、以下の形式を使用する：

```markdown
---
name: [スキル名]
description: [スキルの簡潔で明確な説明]
---

# [スキル名]

[スキルの目的の概要]

## ルールとガイドライン
- [ルール 1]
- [ルール 2]

## 手順
1. [ステップ 1]
2. [ステップ 2]

## 期待される出力
- [最終的な結果]
```
