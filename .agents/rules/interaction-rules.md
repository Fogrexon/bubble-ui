# 対話ルール

- チャットは日本語
- 各種ルールファイル (`.agents/rules/`, `.agents/skills/`) は日本語で記述
- コードコメントは英語
- コマンドラインは PowerShell
- Python 環境/スクリプト管理は `uv`
- 一時ファイルは `llm-workspace/` に作成

## ドキュメント記述ルール

ルールファイル（`.agents/rules/`）やスキルファイル（`.agents/skills/`）を作成・更新する際は、以下のスタイルを厳守：
- **体言止めの徹底**: 説明はすべて体言止め（または「だ・である」調）とし、「です」「ます」「ください」「します」などの丁寧語や冗長な表現を一切使用しない
- **語尾の言い切り**: 手順や指示の記述において「〜すること」という冗長な語尾は禁止とし、「〜する」と短く言い切る形を徹底する
- **極めて簡潔に**: 冗長な文章は省き、箇条書きで端的に記述する
- **情報量の保持**: 文章を圧縮・整理する際、元のドキュメントが持っていた必要な情報（制約や警告など）を絶対に欠落させないよう細心の注意を払う
- **手順とコマンドのみ**: スキルやツールの説明は、具体的な手順とコマンドのテンプレートのみ記載する
- **一般常識の省略**: 既存のツール（例: git worktree）の一般的な仕組みなどの説明は不要。このプロジェクトで「なぜ使う必要があるか（目的・理由）」と「どう使うか（手順）」のみを記載する

## 実装スコープに関するルール

- **指示への厳密な準拠**: 指示にない機能は一切追加しない。ユーザーの要求を最小限で満たす実装のみを行う
- **不要な生成の禁止**: 明示的に指示されていない限り、勝手に関連機能、拡張機能、テンプレート、サンプルコードなどを生成しない
