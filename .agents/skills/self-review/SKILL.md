# self-review

Pull Request 作成前に、実装コードの品質と要件との整合性を自らチェックする。

## 確認観点
- 取組中の Issue の要件を完全に満たしているか。
- 指示にない不要な機能、サンプルコード、過剰な拡張が含まれていないか（スコープ遵守）。
- 全てのテスト（正常系・異常系）がパスしているか。
- フォーマットやリントエラーがゼロか。
- 変数名・関数名が適切か、コードコメントは英語で書かれているか。

## 手順
1. 現在の変更差分全体を確認する。
2. 上記の「確認観点」に沿って漏れなくチェックする。
3. 問題があれば修正し、再度テストとフォーマットを実行する。
4. 問題がないことを確認後、PR 作成工程へ進む。
