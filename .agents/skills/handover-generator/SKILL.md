# handover-generator

LLMのコンテキスト（会話履歴）がリセットされた場合でも、中断箇所から正確に作業を再開できるようにするための引継ぎドキュメントを生成する。

## ドキュメントに含めるべき内容
引継ぎドキュメントには、以下のすべての項目を事細かに漏れなく記述する。

1. **現在の目的・ゴール**
   - 当初の要求事項と、現在目指している最終的なゴール
2. **現在の進行フェーズと状況**
   - ワークフロー体系（例：`task-implementer`）のどのステップまで完了しているか
   - 現在着手中の具体的な作業内容は何か
3. **タスク（TODO）の一覧**
   - 完了済みのTODOリスト（直近でどこまで終わっているか）
   - これから何を行う予定だったのか（未完了のTODOリストを具体的に記述）
4. **作業環境・ブランチ情報**
   - 現在使用している `git worktree` のパス
   - 作業中のブランチ名
5. **変更ファイルと調査結果ドキュメント**
   - どのスクリプトのどの処理を変更・追加しようとしているか
   - 今後の実装で密に関わるパス付きのファイルリスト
   - 初期調査や詳細調査の結果をまとめた既存のドキュメントがあれば、そのパスと概要
6. **発生していた問題や懸念事項**
   - 解決中のエラー、未検証の箇所、後回しにしているリファクタリングなど
7. **次の一手**
   - 会話再開後、LLMが直ちに実行すべき「最初のコマンド」や「次の具体的なアクション」

## 手順
1. 現在の状況（ファイル変更差分、完了済みタスク、未完了タスク）を確認・整理する
2. 上記の「ドキュメントに含めるべき内容」を網羅した引継ぎドキュメントを作成する
   - 保存先: `llm-workspace/` ではなく `llm-docs/` 配下に格納する
   - 命名規則: `llm-docs/xxxx-[ドメイン名]/yyyy-[ドキュメント名]` に則る
3. ユーザーに対し、引継ぎドキュメントの出力パスを報告して作業を一時停止する
