# test-impl

機能実装の前に、要求を満たす検証用のテストを設計・実装する。

## 手順
1. **インターフェースの検討**
   - 既存コードから新規機能をどう呼び出すか（関数名、引数、戻り値等）を決定する。
2. **テストケースの洗い出し**
   - 要件を満たすための正常系・異常系・境界値のケースをリストアップする。
3. **テスト実装**
   - 設計したインターフェースに対するテストコードを記述する。
4. **テストの失敗確認**
   - 本体実装の前にテストを実行し、テストが意図通りに失敗（Red）することを確認する。
